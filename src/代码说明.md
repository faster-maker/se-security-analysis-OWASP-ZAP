### 源码分析说明
在本次 OWASP ZAP 开源安全软件工程实践分析过程中，未直接使用完整项目源码，而是基于分析目标从 ZAP 核心模块中提取关键代码片段，整理后统一存放于仓库 `src/` 目录下。

所选代码片段覆盖以下核心模块，能够精准反映系统核心功能的实现机制：
| 核心模块       | 代码片段内容                                                                 | 存放路径                |
|----------------|------------------------------------------------------------------------------|-------------------------|
| 代理层         | 本地代理配置、HTTPS 证书拦截与转发核心逻辑                                   | `src/proxy-core/`       |
| 扫描引擎       | 主动扫描策略、漏洞检测规则（SQL 注入/XSS）、警报生成核心代码                 | `src/scan-engine/`      |
| 爬虫模块       | 传统爬虫/ AJAX 爬虫的站点遍历、链接解析、参数提取逻辑                       | `src/spider-module/`    |
| API 模块       | 外部调用接口定义、扫描任务提交与结果返回的核心方法                           | `src/api-module/`       |

这种提取关键代码的方式具备以下优势：
1. **避免冗余**：剔除与核心功能无关的辅助代码（如日志、UI 渲染等），降低分析复杂度；
2. **突出重点**：聚焦代理、扫描、爬虫等核心功能的实现逻辑，精准匹配本次分析目标；
3. **提升可读性**：精简后的代码片段更易理解，结合报告中的注释与分析，可快速掌握 ZAP 核心机制；
4. **增强针对性**：所有代码片段均服务于「系统功能建模」「核心设计复原」等核心任务，无无关内容干扰。

`src/` 目录下所有代码片段均标注来源（ZAP 源码包路径、类名），并附关键方法注释，可直接关联报告中的功能分析与 UML 建模内容，保证分析过程的可验证性与严谨性。
